痛点洞察

浏览器里的重复操作，正在消耗你的精力

大量日常业务流程发生在浏览器中——查询、筛选、导出、填写、提交…… 这些操作本身不复杂，但重复性高、记忆成本大、人工容易疲劳。

⚠

重复性强

很多流程每天、每周或每次项目交付都会重复执行，人工操作价值低但耗时稳定存在，浪费大量精力在机械性点击上。

⚗

流程记忆成本高

操作者需要记住入口、页面路径、按钮位置、筛选条件和结果校验方式，换人后还需重新培训，知识难以传承。

⚙

人工执行容易疲劳

重复手动点击、输入、跳转，容易出现遗漏、误点、忘记步骤等问题，导致数据不准、流程中断、需要返工。

⚡

自动化门槛高

传统自动化需要写脚本、维护选择器、理解浏览器控制工具，对非工程用户极不友好，学习成本高昂。

★

经验难以资产化

大量流程只存在于人的操作习惯里，没有形成标准化、可复用、可被 AI 执行的 Skill，经验随人流失。

解决思路

演示一次，永久复用

Op2Skill 的核心思路是——让用户在内嵌浏览器中正常操作一次， 系统自动记录浏览器事件，并将这些操作行为转化为 dodo 可使用的 Skill。

核心功能

强大而直观的录制能力

从事件录制到 Skill 生成，Op2Skill 提供了一站式的浏览器操作沉淀体验。

◆

浏览器事件录制

记录页面导航、点击、输入、键盘、滚动、复制粘贴、文件下载等关键操作，完整捕获元素信息（CSS Selector、XPath、ARIA、文本内容）。

☰

录制历史管理

多轮录制持久化存储，以时间戳自动命名，支持重命名、删除、快速切换查看。所有数据写入本地文件系统，重启不丢失。

⌚

实时事件时间轴

录制过程中实时展示事件列表和时间轴，事件密度分布一目了然，用户可以清晰了解当前捕获进度。

⚄

JSON 预览与导出

录制数据结构化为 events.json，支持完整的结构化数据展示与文件导出，方便调试、归档或作为后续 Skill 生成输入。

✎

Skill.md 生成

调用工作流将结构化事件转化为 dodo 可执行的 skill.md，支持界面内 Markdown 渲染预览，保留原始输出供调试比较。

📄

Skill 编辑与导出

生成的 Skill 可在界面中预览、手动编辑，支持导出为 Markdown 文件，便于沉淀到团队知识库或 Skill 仓库。

⚚

浏览器调试工具

内置缩放控制、清除缓存与 Cookie、Cookie 管理（查看/添加/编辑/删除）、密码管理（查看/添加/编辑/删除已保存凭据）。

数据报表自动化导出

每天需要登录企业数据后台，手动选择统计周期、勾选业务分类、设置筛选条件、批量勾选数据后导出 Excel。整套操作路径完全固定，每天耗时十几分钟，还经常漏选错筛。使用 Op2Skill 录制一次，后续 dodo 自动完成全部筛选导出，无需手动重复操作。

" 💻 研发 / 测试

本地项目功能回归测试

每次本地迭代后，手动访问 localhost 测试项目，完成登录、侧边栏跳转、表单输入、按钮点击、详情页校验等固定操作。借助 Op2Skill 录制一次完整流程生成 Skill，后续版本迭代后一键自动跑完整套功能回归，省去人工重复测试成本。

" 📝 项目 / 产品

每周自动写周报并提交

每周登录周报系统，填写本周完成内容、下周工作计划、风险事项，页面路径和固定内容很相似，每周重复录入十分耗费精力。通过 Op2Skill 录制一次全流程并生成 Skill，每周一键运行自动填充周报，仅需微调少量动态数据即可。

产品创新

不止是录屏工具

Op2Skill 重新定义了浏览器自动化操作的创建方式。

✓

从「写脚本」到「演示一次」

传统自动化要求用户理解脚本、选择器和浏览器 API；Op2Skill 将创建方式变成自然操作，让所有人都能做自动化。

✓

隐性经验→可执行资产

原本只存在于人经验里的业务流程，通过录制和 Skill 生成沉淀为标准化文件，便于复用、传播和维护。

✓

面向 AI Agent 的流程采集入口

不只是录屏工具，而是面向 dodo 自动化能力建设的 Skill 生产工具，让真实用户操作成为 Agent 能力构建的数据来源。

支持多轮录制持久化存储与管理，随时切换查看不同轮次的录制结果和生成的 Skill，构建可积累的操作资产库。
